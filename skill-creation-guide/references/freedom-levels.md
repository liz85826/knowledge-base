# 自由度三级模式详解

技能包的执行指令应根据任务特征选择合适的自由度级别。

---

## 高自由度（文字说明）

**适用场景**：多种方案均可行，决策依赖上下文，用启发式规则引导

```markdown
## 旋转 PDF
识别用户要旋转的角度（90/180/270度），
使用 scripts/rotate_pdf.py 执行，
完成后告知用户处理了多少页。
```

**特征**：
- Claude 有充分空间自主判断
- 描述"做什么"，不限制"怎么做"
- 适合创意类、分析类、开放性任务

---

## 中等自由度（伪代码 + 参数声明）

**适用场景**：有首选模式，允许适度变化，配置参数影响行为

```markdown
## 输入参数
- file_path: str      # 必填，绝对路径
- rotation: int       # 必填，值为 90 / 180 / 270
- pages: list[int]    # 可选，默认处理全部页面

## 执行逻辑
if pages 为空:
    处理全部页面
else:
    仅处理指定页码（从1开始）

调用 scripts/rotate_pdf.py，传入上述参数

## 输出
- 成功：返回新文件路径 + 处理页数
- 失败：返回错误类型 + 具体原因（不修改原文件）
```

**特征**：
- 明确输入输出契约
- 核心逻辑固定，边缘情况允许灵活处理
- 适合数据处理、格式转换类任务

---

## 低自由度（具体脚本 + 严格步骤序列）

**适用场景**：操作脆弱易出错，一致性至关重要，必须严格按序执行

```markdown
## 执行步骤（严格按序，不得跳过）

1. **验证输入**
   - 检查 file_path 文件存在
   - 检查文件扩展名为 .pdf
   - 检查 rotation 值为 90/180/270 之一
   - 任意验证失败 → 立即报错，不执行后续步骤

2. **备份原文件**
   - 复制到 `{file_path}.bak`
   - 确认备份文件大小与原文件一致

3. **执行旋转**
   ```bash
   python scripts/rotate_pdf.py "{file_path}" {rotation}
   ```

4. **验证结果**
   - 输出文件大小 > 0 字节
   - 输出文件可被 PDF 库打开

5. **返回结果**
   - 成功：告知新文件路径
   - 失败：保留 .bak 备份，报告具体错误

## 错误处理
| 错误类型 | 处理方式 |
|---------|---------|
| 文件不存在 | 提示检查路径，终止 |
| 角度不合法 | 列出合法值，重新询问 |
| 处理失败 | 保留原文件，报告错误 |
```

**特征**：
- 每步骤都有明确的前置条件和后置验证
- 错误处理覆盖所有已知失败场景
- 适合文件操作、金融计算、数据库写入等高风险任务

---

## 选择指南

```
任务是否有明确的"唯一正确执行序列"？
  ├── 是 → 低自由度（严格步骤）
  └── 否 → 任务失败代价大吗？
            ├── 大（数据损坏/财务错误）→ 低自由度
            └── 一般 → 是否有首选方案？
                        ├── 有 → 中等自由度（参数化）
                        └── 无 → 高自由度（文字说明）
```
